# 可配置账号购买页设计

## 目标

把用户端“账号购买”页面从前端写死改为系统设置驱动。站长可以在后台表单里修改 QQ、页面文案和商品信息，也可以新增 Gemini、Claude、住宅 IP 等更多商品；保存后刷新前台立即生效，不需要重新构建 Docker 镜像。

## 后台配置

在“系统设置”的通用设置页增加“账号购买设置”表单区，保持现有 settings 表单风格。配置存入 `settings.account_store_config` JSON 字段。

页面级字段：

- 是否启用账号购买页。
- 页面标题。
- 页面说明。
- 右侧状态文案。
- 联系方式类型，第一版支持 QQ、微信、邮箱和自定义。
- 联系方式内容。
- 联系区提示文案。
- 复制按钮文案。
- 底部免责声明。

商品字段：

- 是否启用。
- 商品名称。
- 副标题说明。
- 价格。
- 币种。
- 单位。
- 标签。
- 图标类型：OpenAI、Gemini、手机、Wi-Fi、邮箱、钥匙、地球。
- 主题色：青绿、黄色、蓝色、紫色、灰色。
- 功能点输入框。用中文逗号 `，` 或英文逗号 `,` 分隔，保存时拆成数组，前台逐行展示。
- 风险提示，可留空。
- 排序。后台支持上移、下移，前台按顺序展示。

后台支持新增、复制、删除商品。删除和关闭商品只影响前台展示，不影响历史配置或其他业务。

## 用户端展示

`/account-store` 继续使用现有白色为主的卡片样式。桌面默认每行 3 个商品，超过 3 个自动换行；中屏每行 2 个，手机每行 1 个。只展示启用商品。

如果后台没有配置或配置损坏，前台回退到当前默认三项：OpenAI 成品账号、长效手机号接码、美国静态住宅 IP，避免页面空白。

## 数据流

后端通过管理员 `/api/v1/admin/settings` 读取和保存完整配置；通过公开 `/api/v1/settings/public` 暴露给登录后的前台页面。配置是展示信息，不包含密钥或敏感凭据。

## 验证

- 后端测试覆盖公开设置和管理员设置会返回 `account_store_config`。
- 前端测试覆盖商品配置渲染、逗号拆分功能点、新增商品表单、复制联系方式。
- 构建通过后部署到 VPS，验证 `/account-store` 和 `/health`。
