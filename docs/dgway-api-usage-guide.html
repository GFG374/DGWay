先理解四层关系

你刚才看到账号编辑页里有 gpt-image-2，这个判断是对的：它说明这个 OpenAI 账号本身支持这个模型。 但 Sub2API 真正调用时还会经过下面几层，所以“账号有模型”不等于“任意 API Key 都能直接调用”。

层级 你在哪里看 作用 常见问题
代理 IP管理 让服务器出口走日本/新加坡节点，解决 OpenAI 地区限制。 代理不通会导致授权 403 或上游连接失败。
账号 账号管理 -> 编辑账号 保存 OpenAI OAuth 凭证、可用模型白名单、并发、代理。 模型白名单没有的模型会调度失败。
分组 系统分组或后台分组设置 控制 API Key 能调度哪些账号，以及是否允许图片生成。 图片权限没开会返回 Image generation is not enabled for this group。
API Key API 密钥 客户端真正使用的密钥，必须绑定到正确分组。 Key 没分组或分组不对，会无法调度账号。
重点： 账号页看到 gpt-image-2 是账号能力；图片接口能不能调用，还要看 API Key 绑定的分组是否开启图片生成。

在哪里看可用模型

方法一：后台账号页

进入 账号管理 -> 编辑账号 -> 模型限制。你截图里的模型白名单就是账号当前可用模型。

gpt-5.4-mini gpt-5.4 gpt-5.5 gpt-5.2 gpt-5.2-pro gpt-5.3-codex gpt-image-1 gpt-image-1.5 gpt-image-2

方法二：调用模型列表接口

这是客户端侧最准确的方法。把 sk-你的密钥 换成你自己创建的 DGWay API Key。

curl http://8.148.191.211/v1/models \
  -H "Authorization: Bearer sk-你的密钥"
不要拿 OpenAI 官网模型名硬试。你当前账号能调度什么，以后台模型白名单和 /v1/models 返回为准。 例如 gpt-4o-mini 不在当前列表里，所以会失败；先用 gpt-5.4-mini。

创建 API Key

  1. 左侧进入 API 密钥。
  2. 点击创建密钥，名称可以填 我的测试 Key。
  3. 分组选择 openai-default。这是当前 OpenAI OAuth 账号所在分组。
  4. 创建后马上复制 sk-...。密钥通常只完整显示一次。
  5. 客户端里使用 http://8.148.191.211/v1 作为 Base URL。
API Key 不要发给别人，也不要写进公开仓库。测试 Key 用完可以在后台删除或禁用。

文本调用测试

最稳的文本模型先用 gpt-5.4-mini。下面这个请求成功时，返回内容应该只包含 DGWay OK。 

curl http://8.148.191.211/v1/chat/completions \
  -H "Authorization: Bearer sk-你的密钥" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.4-mini",
    "messages": [
      {
        "role": "user",
        "content": "Reply with only: DGWay OK"
      }
    ],
    "max_tokens": 32
  }'

Responses API 测试

一些工具或 Codex 风格客户端会走 /v1/responses，也可以这样测：

curl http://8.148.191.211/v1/responses \
  -H "Authorization: Bearer sk-你的密钥" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.4-mini",
    "input": "Reply with only: DGWay OK",
    "max_output_tokens": 32
  }'

gpt-image-2 图片测试

你截图里看到 gpt-image-2 是对的。这个模型在账号白名单里，图片生成也已经实测成功。

图片生成请求

curl http://8.148.191.211/v1/images/generations \
  -H "Authorization: Bearer sk-你的密钥" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "Create a clean square app icon for DGWay API: a bold letter D, blue and cyan tech style, simple vector-like design, white background.",
    "size": "1024x1024",
    "n": 1
  }'

本次测试结果

服务器返回了 b64_json，并已解码成 PNG。

如果返回图片权限错误

错误内容通常是：

{"error":{"message":"Image generation is not enabled for this group","type":"permission_error"}}

处理顺序：

  1. 确认 API Key 绑定的是 openai-default 分组。
  2. 确认该分组开启了图片生成权限。
  3. 如果刚改完分组权限，旧 Key 可能吃到旧认证缓存；新建一个 Key 最干净。
  4. 确认账号模型白名单里有 gpt-image-2。

常见错误

现象 原因 处理方法
localhost:1455 打不开 OpenAI OAuth 的本地回调地址，本来就不会在你的浏览器上有服务。 复制完整回调 URL 或 code，粘回 Sub2API 授权输入框。
unsupported_country_region_territory 服务器出口地区不被 OpenAI 支持。 使用 mihomo-japan-local 代理。当前已配置日本节点。
401 API key is required 没有带 DGWay API Key，或 Bearer 格式错。 请求头写：Authorization: Bearer sk-...。
503 no available accounts 没有账号能调度这个模型，常见于模型不在白名单里。 先调 /v1/models 看可用模型，优先用 gpt-5.4-mini。
Image generation is not enabled API Key 绑定分组没有开图片生成。 开启分组图片权限，或创建新的已授权 API Key。
