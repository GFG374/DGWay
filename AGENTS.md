# DGWay Agent Rules

这些规则用于后续所有 Agent、Codex、Claude Code 或自动化会话。修改 DGWay API 时必须遵守。

## Git 同步

- 每一轮有效代码或文档修改完成后，都要提交并推送到 GitHub 主分支。
- 推送前必须先运行和本轮改动相关的测试或构建，并把结果记录在最终回复里。
- 不要把临时密钥、Cookie、OAuth token、数据库密码、二维码、测试输出目录提交到 Git。
- 工作区里有无关改动时，只能显式 `git add` 本轮相关文件。

## 服务器部署

- 禁止在服务器上创建长期二进制备份文件，例如 `sub2api.bak.*`。
- 如需回滚，优先使用 Git 提交、重新构建和重新部署，不依赖服务器本地备份。
- 部署后检查 `sub2api.service` 是否为 `active`，并验证关键接口。
- 不要把构建产物、依赖目录或临时扫描结果长期留在服务器上。

## 文档

- 用户可见流程变更要同步更新 `docs/DGWAY_USAGE_CN.md`。
- 商业化、额度、分组和风险判断要同步更新 `docs/DGWay-API-设置与盈利.md`。
- 文档用简单中文写，优先写用户应该填什么、在哪里填、报错时检查什么。

## 安全

- 真实 API Key 只能用于临时测试，回复和文档中必须打码。
- 发现密钥泄露、误提交或公开展示时，先提示删除/轮换密钥，再继续工作。
- 不要扩大用户权限、开放注册或调整支付配置，除非用户明确要求。
